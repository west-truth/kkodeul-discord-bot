# 꼬들봇 (Kkodeul Bot)

한국어 워들(Wordle) 게임인 '꼬들'을 디스코드 서버 내에서 바로 즐길 수 있도록 만들어진 봇입니다. 

## 📖 게임 규칙 (일반 꼬들과의 차이점)

*   총 **6개의 자음과 모음**을 이용해서 숨겨진 단어를 맞추는 게임입니다.
*   단어는 항상 **풀어쓰기** 방식으로 입력됩니다. (예: `꿀벌` -> `ㄲ ㅜ ㄹ ㅂ ㅓ ㄹ`)
*   **자모음 인정 기준 (키보드 배열 기준):**
    *   한 번에 입력 가능한 글자는 하나로 인정됩니다. (`ㄲ`, `ㄸ`, `ㅃ`, `ㅆ`, `ㅉ`, `ㅐ`, `ㅒ`, `ㅔ`, `ㅖ` 등)
    *   복합 모음이나 겹받침은 두 개의 자모로 분리됩니다. (예: `ㅘ` -> `ㅗ` + `ㅏ` / `슭` -> `ㅅ` + `ㅡ` + `ㄹ` + `ㄱ`)
*   **결과 색상의 의미:**
    *   🟩 **초록색**: 자모가 정답에 포함되어 있고 위치도 완벽히 일치함
    *   🟨 **노란색**: 자모가 정답에 포함되어 있지만 위치가 틀림
    *   ⬛ **회색**: 자모가 정답에 포함되어 있지 않음

## 🎮 주요 기능 및 명령어

*   `/꼬들`: 매일 00시에 새롭게 갱신되는 '오늘의 단어'를 맞추는 일일 과제 모드입니다. 하루에 한 번만 참여 가능합니다.
*   `/무제한`: 꼬들 게임을 횟수 제한 없이 즐길 수 있는 모드입니다. 게임 진행 상황은 채널에 모두 공개됩니다.
*   `/대결 [@상대방]`: 지정한 상대방 유저와 1:1로 꼬들 대결을 진행합니다. 먼저 정답 단어를 맞추는 플레이어가 승리합니다.
*   `/추측 [단어]`: (게임 진행 중) 정답을 추측하여 제출합니다. (예: `/추측 사과` 또는 `/추측 ㅅㅏㄱㅗㅏ`)
*   `/리더보드`: 유저들의 최대 연속 성공 횟수를 기준으로 랭킹 순위를 보여줍니다.

## 🛠 사용 기술 및 라이브러리

*   **Node.js**: 런타임 환경
*   **discord.js (v14)**: Discord API 연동 및 슬래시(/) 명령어 처리
*   **canvas**: 게임 진행 상황을 직관적인 이미지로 렌더링
*   **hangul-js**: 꼬들 게임의 핵심인 한글 자모 분리 및 조립 로직 처리
*   **node-cron**: 매일 자정에 '오늘의 꼬들' 단어를 갱신하는 타이머 스케줄링

## 📁 주요 디렉토리 및 구조

*   `index.js`: 봇을 실행하는 메인 파일입니다. 명령어 등록, 상호작용 및 이벤트 처리 로직이 포함되어 있습니다.
*   `kkodeul_engine.js`: 단어 자소 분리 및 정답 판별, 진행판 이미지 렌더링 등 핵심 게임 엔진 스크립트입니다.
*   `data_manager.js`: 플레이어의 전적, 통계 및 일일 게임 참가 여부를 `data/` 폴더에 저장/불러오기 하는 관리 파일입니다.
*   `config.json`: **(보안 주의)** 봇 토큰 등 중요 설정이 들어가는 파일입니다. Github에 업로드되지 않도록 `.gitignore`에 등록해야 합니다. `config.example.json`을 복사하여 작성하세요.
*   `data/`: 게임 진행 기록(JSON 포맷)이 저장되는 폴더입니다.
*   `fonts/`: 캔버스 이미지 생성에 필요한 글꼴 파일들이 있습니다.

## 🚀 봇 설치 및 서버 실행 가이드

1.  **리포지토리 클론 및 의존성 패키지 설치**
    ```bash
    git clone https://github.com/your-username/kkodeul-bot.git
    cd kkodeul-bot
    npm install
    ```

2.  **환경 설정 (config)**
    *   보안 유출 방지를 위해 `.env` 파일 등에 봇 토큰 값을 넣는 대신 `config.example.json` 템플릿을 사용합니다.
    *   `config.example.json` 파일의 이름을 `config.json`으로 변경합니다.
    *   변경한 `config.json` 파일에 본인이 발급받은 디스코드 봇 정보를 알맞게 입력하세요.
    ```json
    {
      "token": "본인의_디스코드_봇_토큰_입력",
      "clientId": "봇의_클라이언트(애플리케이션)_ID_입력",
      "guildId": "명령어를_등록할_테스트_서버(길드)_ID_입력"
    }
    ```

3.  **봇 실행하기**
    설정이 끝났다면 아래 명령어로 봇을 켭니다. 구동 시 설정한 테스트 서버에 슬래시 명령어가 자동으로 등록됩니다.
    ```bash
    npm start
    ```

*(선택) 명령어 초기화*: 만약 등록된 전역/로컬 슬래시 명령어들을 완전히 지우고 싶다면 `node cleanup.js` 스크립트를 한 번 실행하시기 바랍니다.
